FENRIR — Network Input Dataset Metadata

File name: network_input.csv

Purpose: This dataset is designed to serve as input for the Interbank
Network & Propagation module. It provides bank-level structural and risk
characteristics used to calibrate the exposure matrix W and simulate
systemic contagion dynamics.

Observational Unit: One row per bank (identified by LEI_Code).

Number of Institutions: Automatically generated from the FENRIR systemic
risk pipeline.

Columns Description:

1.  LEI_Code
    -   Legal Entity Identifier (unique bank identifier).
    -   Used as node ID in the interbank network.
2.  BankName
    -   Official bank name (for interpretability and reporting).
3.  Country
    -   Country of incorporation or supervisory reporting.
4.  TotalAssets
    -   Total assets (in EUR millions, as reported in EBA transparency
        dataset).
    -   Proxy for bank size.
5.  TotalAssets_norm
    -   Min–max normalized TotalAssets in [0,1].
    -   Used for probabilistic calibration of interbank links and edge
        weights.
6.  CR
    -   Capital Ratio (CET1 / Total Assets).
    -   Used to determine vulnerability and default threshold proximity.
7.  M_i_final
    -   Macro Risk Score (hybrid score combining weighted indicators and
        PCA).
    -   Captures exposure to macro/systemic risk factors.
8.  FI
    -   Fragility Index.
    -   Computed as a function of NPL and leverage components.
    -   Proxy for liquidity/fragility risk (not a regulatory liquidity
        ratio).

Intended Use in Network Module:

• Nodes = banks (LEI_Code) • Edge probabilities calibrated using
TotalAssets_norm and/or M_i_final • Initial capital shocks based on
M_i_final • Default propagation driven by CR and simulated capital
erosion

Important Notes:

• This file does NOT contain bilateral exposures. • Exposure matrix W
must be generated separately (e.g., scale-free or calibrated synthetic
network). • This dataset provides structural and risk calibration inputs
only.
